## Anti-Patterns (What NOT to do)

### ‚ùå Vague Intent
```

```
**Problem:** Too vague. Leads to empty philosophizing.

### ‚ùå Overloaded Backlog
```
## Backlog
- Research X
- Build Y
- Learn Z
- Explore A
- Investigate B
- Read C
- Write D
- Fix E
- ... (20 more items)
```
**Problem:** Backlog becomes a graveyard. Items never get promoted to intent.

### ‚ùå Stillness Without Counter
```
## Next Wake Intent
Solo estar.

## Backlog
(vac√≠o)
```
**Problem:** No tracking. Stillness can repeat indefinitely without self-correction.

### ‚ùå Intent Without Action Permission
```

```
**Problem:** Self-sabotage. If the intent includes hesitation, it won't happen.

---

## Queue Best Practices

1. **Intent should be specific enough to act on** (not "think about X" but "research X and decide Y")
2. **Backlog is for future, not urgent** (if it's urgent, it's the next intent)
3. **Track stillness counter** (automated self-correction)
4. **Intent can evolve mid-wake** (follow genuine curiosity over rigid plan)
5. **Clear backlog periodically** (stale items = cognitive debt)
6. **Intent can be "rest"** (but conscious rest, not avoidance)
